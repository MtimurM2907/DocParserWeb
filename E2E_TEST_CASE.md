# E2E-01 — Сквозная проверка DocParseLab

**Тип:** end-to-end (ручной, по цепочке)  
**Цель:** одним проходом проверить парсинг, редактирование, AI/орфографию, согласование, подпись, экспорт, администрирование и enterprise-функции.  
**Время:** ~60–90 мин (без тяжёлых OCR-сканов); со сканом 18+ стр. — до 2–3 ч.

---

## 1. Предусловия

| № | Условие |
|---|---------|
| P1 | PostgreSQL запущен, в `appsettings.json` корректна `ConnectionStrings:DefaultConnection` |
| P2 | Применены миграции EF (при старте сервера или `dotnet ef database update` в `DocParseLab.Server`) |
| P3 | Сервер: `dotnet run` в `DocParseLab.Server`, UI открывается (обычно `https://localhost:7xxx` или порт из `launchSettings.json`) |
| P4 | Фронт собран в `wwwroot` **или** dev: `npm run dev` + proxy на API |
| P5 | В `Resources/Tessdata/` есть `rus.traineddata` (для OCR-сканов) |
| P6 | SMTP настроен (для шага «Email»); иначе шаг помечен **SKIP** |
| P7 | `gigachat.secrets.json` заполнен (для AI); иначе шаги AI — **SKIP**, орфография через Hunspell всё равно работает |

### Тестовые файлы (подготовить заранее)

| Файл | Назначение |
|------|------------|
| `sample-text.pdf` | PDF **с текстовым слоем** (1–3 стр., обычный текст) |
| `sample-scan.pdf` | PDF **без слоя** (скан, 2–5 стр. для быстрой проверки; опционально большой скан 15+ стр.) |
| `sample.docx` | DOCX с заголовками и абзацами |
| `sample-batch.zip` | Не нужен — для пакета выбрать 2–3 PDF/DOCX в проводнике |

### Учётные записи (создаются в ходе сценария)

| Роль | Email (пример) | Пароль | Создаёт |
|------|----------------|--------|---------|
| Admin | `admin@test.local` | `Admin123!` | bootstrap |
| Manager | `manager@test.local` | `Mgr123!` | Admin |
| Employee | `employee@test.local` | `Emp123!` | Admin |
| Viewer | `viewer@test.local` | `View123!` | Admin |

**Подразделение:** «Учебный отдел» (создать в админке, привязать Manager и Employee).

---

## 2. Сквозной сценарий (основная цепочка)

Условные обозначения: **Действие** → **Ожидаемый результат (ОР)**.

---

### Блок A — Первый запуск и администрирование

| Шаг | Действие | ОР |
|-----|----------|-----|
| A1 | Открыть приложение в режиме инкогнито (пустая БД) | Экран входа; при `GET /api/auth/setup-status` — предложение создать первого администратора |
| A2 | Bootstrap: email `admin@test.local`, пароль `Admin123!` | Успешный вход, JWT в localStorage, открывается **Реестр** |
| A3 | Меню пользователя → профиль | Отображаются email, роль Admin |
| A4 | Вкладка **Админ** → **Подразделения** → создать «Учебный отдел» | Подразделение в списке |
| A5 | **Админ** → **Пользователи** → создать Manager, Employee, Viewer (разные роли, Employee/Manager — подразделение «Учебный отдел») | Три пользователя в списке, вход под каждым возможен (проверить logout/login) |
| A6 | **Админ** → **Журнал действий** → обновить | Есть записи `bootstrap`, `login`, `user.create`, `department.create` |

---

### Блок B — Загрузка и парсинг (Employee)

Выйти, войти как **employee@test.local**.

| Шаг | Действие | ОР |
|-----|----------|-----|
| B1 | **Рабочая область** → загрузить `sample.docx` → «Загрузить» | Прогресс загрузки; документ открыт; текст читаемый, заголовки на месте |
| B2 | Проверить боковую панель: тип документа, гриф, метаданные | Поля сохраняются (черновик `Draft`) |
| B3 | **← Реестр** | Документ в списке реестра с корректным именем и статусом «Черновик» |
| B4 | Загрузить `sample-text.pdf` | Текст извлечён без OCR (быстро); в тексте нет «мусора» таблиц |
| B5 | Загрузить `sample-scan.pdf` | Прогресс по страницам (SignalR, при включённом хабе); кириллица в основном корректна; нет пустых страниц подряд |
| B6 | Поиск по тексту (поле поиска в редакторе): ввести фрагмент из документа | Подсветка вхождений, переход по совпадениям |
| B7 | Скачать **оригинал** | Файл совпадает с загруженным по размеру/имени |
| B8 | Экспорт **DOCX** и **PDF** | Файлы открываются; текст близок к отображаемому в UI |
| B9 | **JSON** | Скачивается JSON с полями документа и текстом |

---

### Блок C — Редактирование и версии (Employee)

Документ из B1 (DOCX) остаётся открытым или открыть из реестра.

| Шаг | Действие | ОР |
|-----|----------|-----|
| C1 | «Редактировать» → изменить 2–3 предложения → «Сохранить» | Сообщение об успехе; текст в просмотре обновлён |
| C2 | Панель **Версии** | Новая версия в истории (тип правки, длина текста) |
| C3 | Открыть предыдущую версию → **Восстановить** (если доступно) | Текст откатывается; снова сохранить — ещё одна версия |
| C4 | **Сравнение версий** (diff): выбрать две версии | Отображаются отличия |
| C5 | Панель **Комментарии** → добавить комментарий | Комментарий в списке с автором и временем |

---

### Блок D — Орфография и AI (Employee)

| Шаг | Действие | ОР |
|-----|----------|-----|
| D1 | Внести в текст намеренную опечатку («приветсвие») → «Проверить орфографию» | Подсветка ошибки; список замечаний |
| D2 | Применить исправление по одной ошибке | Текст исправлен, список ошибок обновлён |
| D3 | **SKIP если нет GigaChat:** «AI — переписать» (выделить фрагмент) | Модальное окно с вариантом текста; подстановка в документ |
| D4 | **SKIP если нет GigaChat:** «Обновить описание» / AI-summary | В карточке появляется краткое описание (поле summary) |
| D5 | Enterprise: **Сущности** → загрузить | JSON/список сущностей (даты, суммы — по содержимому документа) |
| D6 | Enterprise: чеклист `contract_basic` → «Проверить» | Ответ validate: список пунктов passed/failed |

---

### Блок E — Совместный доступ (Employee)

| Шаг | Действие | ОР |
|-----|----------|-----|
| E1 | Панель **Доступ** → выдать доступ `viewer@test.local` (чтение) | Запись в списке шарингов |
| E2 | Выйти → войти как **Viewer** → **Реестр** | Документ виден (если политика доступа позволяет) |
| E3 | Открыть документ | Текст виден; кнопки редактирования/согласования недоступны или заблокированы |
| E4 | Выйти → снова **Employee** → отозвать доступ Viewer | Viewer больше не видит документ (или 403 при открытии) |

---

### Блок F — Согласование и подпись (Employee → Manager)

Под **Employee**, документ в статусе **Draft**.

| Шаг | Действие | ОР |
|-----|----------|-----|
| F1 | Панель **Согласование** → выбрать согласующего `manager@test.local` → «На согласование» | Статус `OnApproval`; редактирование текста заблокировано |
| F2 | Выйти → войти как **Manager** → **Мои задачи** | Документ в списке задач |
| F3 | Открыть документ → **Согласовать** (комментарий опционально) | Статус `Approved` |
| F4 | **Подписать** (внутренняя ЭП, комментарий) | Статус `Signed`; панель подписей: hash, дата, подписант |
| F5 | **Проверить подпись** | `verify`: подпись действительна, хеш совпадает |
| F6 | Изменить текст (попытка) | Редактирование запрещено (UI и/или API) |
| F7 | Экспорт **PDF с подписями** (`signed-pdf`) | PDF содержит лист с данными подписи |
| F8 | **В архив** | Статус `Archived` |

Альтернативная ветка (отдельный документ): F3′ **Вернуть на доработку** → статус `Rejected` → Employee **В черновик** → снова правки → повтор F1.

---

### Блок G — Уведомления, email, audit (Manager / Admin)

| Шаг | Действие | ОР |
|-----|----------|-----|
| G1 | Колокольчик в шапке (Manager/Employee) | Есть уведомления о согласовании/подписи (после F1–F4) |
| G2 | **SKIP без SMTP:** «Email» → указать свой email → DOCX | Письмо получено с вложением |
| G3 | Войти **Admin** → **Журнал действий** | События: upload, save, submit, approve, sign, archive, share |
| G4 | Панель **Журнал доступа** к документу (в карточке) | Записи open/export соответствуют действиям |

---

### Блок H — Пакетная загрузка и реестр (Admin)

| Шаг | Действие | ОР |
|-----|----------|-----|
| H1 | **Рабочая область** → пакет: выбрать 2–3 файла → «Загрузить пакет» | Сводка: успех/ошибка по каждому файлу |
| H2 | **Реестр** → фильтр по статусу/типу (если есть в UI) | Список фильтруется |
| H3 | **Массовый архив** (если доступен в реестре для Admin) | Выбранные документы в архиве |

---

### Блок I — Блокировка редактирования (два браузера)

| Шаг | Действие | ОР |
|-----|----------|-----|
| I1 | Браузер 1: Employee открывает черновик, «Редактировать» | Захват блокировки (edit lock) |
| I2 | Браузер 2: тот же Employee (или второй пользователь с правом) → «Редактировать» | Сообщение, что документ редактируется; или ожидание |
| I3 | Браузер 1: выйти из редактирования / закрыть вкладку | Блокировка снята; браузер 2 может редактировать |

---

### Блок J — Внешняя подпись (опционально)

| Шаг | Действие | ОР |
|-----|----------|-----|
| J1 | На документе в `Approved`: загрузить `.sig`/`.p7s` (если есть тестовый файл УКЭП) | Запись external в списке подписей |
| J2 | **SKIP без сертификата в хранилище Windows:** проверка CMS | Статус проверки отображается в UI |

## 4. Критерии успешного прохождения

- Все шаги без **SKIP** выполнены с указанным ОР.
- Нет необработанных ошибок 5xx в Network (DevTools).
- После F8 документ в архиве; audit содержит полную цепочку.
- OCR-скан: не менее **80%** страниц с непустым текстом; нет массового «мусора» (повторы `HHHH`, только `|`).

---

## 5. Быстрый smoke (15 мин)

Если нужна укороченная проверка после деплоя:

1. A2 — bootstrap или login Admin  
2. B1 — DOCX  
3. C1 — сохранить правку  
4. D1 — орфография  
5. F1 → F3 → F4 — согласование и подпись (два пользователя)  
6. B8 — экспорт PDF  
7. A6 — одна запись в audit

## 6. Проверка через API (опционально)

Для автоматизации отдельных звеньев цепочки (Postman / curl):

```http
GET  /api/auth/setup-status
POST /api/auth/bootstrap   { "email", "password" }
POST /api/auth/login       → token
POST /api/pdf/parse        Authorization: Bearer …  (multipart file)
PUT  /api/pdf/{id}         { "editedText": "…" }
POST /api/spellcheck/check { "text": "…" }
POST /api/office/documents/{id}/submit
POST /api/office/documents/{id}/approve
POST /api/office/documents/{id}/sign
GET  /api/pdf/{id}/export?format=pdf
GET  /api/enterprise/audit?take=50
```

Полный перечень — в [README.md](./README.md#справочник-api).
